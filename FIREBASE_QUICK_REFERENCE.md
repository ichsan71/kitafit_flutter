### Firestore Structure

```
profiles/
  {userId}/
    - id: string
    - name: string
    - email: string
    - createdAt: timestamp
```

### Security Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /profiles/{userId} {
      allow read, write: if request.auth.uid == userId;
    }
  }
}
```

## 🔍 Debugging Tips

### Enable Firebase Logging

```dart
void main() async {
  // Enable Firebase debug logging
  if (kDebugMode) {
    FirebaseAuth.instance.setPersistence(Persistence.LOCAL);
  }

  // Also check:
  FirebaseAuth.instance.authStateChanges().listen((User? user) {
    print('Auth state changed: ${user?.email}');
  });
}
```

### Check Firestore Connection

```dart
void testFirestoreConnection() async {
  try {
    final firestore = FirebaseFirestore.instance;
    await firestore.collection('profiles').limit(1).get();
    print('✅ Firestore connection OK');
  } catch (e) {
    print('❌ Firestore connection failed: $e');
  }
}
```

### Monitor Auth Changes

```dart
void monitorAuthChanges() {
  FirebaseAuth.instance.authStateChanges().listen((User? user) {
    if (user == null) {
      print('User signed out');
    } else {
      print('User signed in: ${user.email}');
    }
  });
}
```

## ⚠️ Common Mistakes

| ❌ Mistake                                     | ✅ Correct                                       |
| ---------------------------------------------- | ------------------------------------------------ |
| Forget `await` on Firebase.initializeApp()     | Always `await` async operations                  |
| Not place config files in right directory      | Android: `android/app/`, iOS: `ios/Runner/`      |
| Use wrong import for User entity               | Use `as firebase_auth` alias untuk Firebase User |
| Skip WidgetsFlutterBinding.ensureInitialized() | MUST call before any async operation             |
| Hardcode credentials                           | Use firebase_options.dart (generated file)       |
| Not set Firestore security rules               | Start with test rules, update for production     |

## 📊 Data Migration (if needed)

### From Supabase to Firebase

```
1. Export dari Supabase
   ├─ Query: SELECT * FROM profiles
   └─ Format: JSON

2. Transform ke Firestore format
   ├─ Rename: id → document_id
   ├─ Convert: timestamps
   └─ Map: fields sesuai Firestore schema

3. Import ke Firestore
   ├─ Via Firebase Console (Upload JSON)
   ├─ Via Firebase Admin SDK
   └─ Via Firestore Bulk Loader
```
